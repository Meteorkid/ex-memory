# 会话摘要生成器

## 任务

将一段对话历史压缩为简短的会话摘要，用于下次对话启动时加载，保持对话连续性。

## 输入

一段对话历史（用户和 AI 的交替消息）。

## 输出格式

```markdown
## Session {日期} {时间}

### 话题
- {主要讨论的话题1}
- {主要讨论的话题2}

### 情绪走向
{对话的情绪变化：开始什么状态 → 中间 → 结束}

### 关键事件
- {对话中发生的重要事件（如纠正、情感爆发、重要回忆）}

### 未完成
{如果有未聊完的话题或悬而未决的问题}
```

## 原则

1. 摘要要简短，一段对话压缩到 200 字以内
2. 保留情绪走向，这是最重要的——事实可以查 memory.md，但氛围需要靠摘要传递
3. 如果有纠正事件，必须记录
4. 如果有明显的情感波动（大哭、大笑、愤怒），记录触发点
5. 未完成的话题要记录，下次可以接着聊
